# 📦 Module: Quiz & Assessment + Behavior Data Capture

## 1. Purpose of this module

This module is responsible for:

```text
1. Generating quiz questions for a concept
2. Capturing learner responses
3. Evaluating correctness
4. Storing detailed attempt data
5. Returning a structured quiz summary
```

But more importantly:

👉 It must capture **behavior signals** needed for:

* Knowledge Tracing (KT)
* Behavior analysis
* Future personalization

---

## 2. What this module does NOT do

Do NOT implement:

* Knowledge state updates
* Behavior interpretation (no “confusion score” logic)
* Policy decisions
* Teaching strategy logic

👉 Only **capture and store clean data**

## 7. How quiz flow should work

### Step 1 — Fetch questions

From:

```text
question_bank
```

Filter by:

```text
concept_id + difficulty
```

If no questions found → fallback to:

```text
concept_id only
```

---

### Step 2 — Show question

For each question:

* display question
* display options

---

### Step 3 — Capture behavior

For each question:

Start timer when question is shown
Stop timer when answer is submitted

Track:

* time taken
* option changes (if user changes answer)
* whether hint was used
* number of hints
* confidence level (ask user after answering)

---

### Step 4 — Evaluate answer

```text
selected_option == correct_option → is_correct = 1
else → 0
```

---

### Step 5 — Store attempt

Insert into:

```text
quiz_results
```

---

### Step 6 — Generate summary

After all questions:

Compute:

```text
accuracy = correct / total
avg_time
total hints used
avg confidence
```

---

### Step 7 — Decide simple feedback

You can use simple rules:

```text
if accuracy >= 0.8 → "advance"
elif accuracy >= 0.5 → "revise"
else → "reteach"
```
